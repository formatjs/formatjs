# Repo Layout

## Directory Structure

```
formatjs/
  packages/           # 33 npm packages (polyfills, parsers, integrations, tools)
  crates/             # 3 Rust crates (CLI, parser, skeleton parser)
  tools/              # Custom Bazel macros (ts_compile, vitest, oxc_transpiler, etc.)
  docs/               # Documentation site (Vike + React + MDX)
  benchmarks/         # Cross-tool benchmarks (Rust CLI vs TS CLI)
  conformance-tests/  # ICU4J conformance testing
  knowledge-base/     # Architecture docs and migration plans
  patches/            # pnpm patches for upstream deps
  .github/workflows/  # CI/CD (test, release, rust-cli-release, website)
  .aspect/bazelrc/    # Aspect-recommended Bazel settings
  sg-rules/           # ast-grep structural lint rules
```

## Package Management: pnpm

**Version:** pnpm 10.4.1 (pinned in MODULE.bazel), 10.32.1 (root package.json)

- `pnpm-workspace.yaml` defines workspace packages
- `npm_translate_lock` in MODULE.bazel converts `pnpm-lock.yaml` to Bazel deps
- pnpm patches in `patches/` for upstream compatibility
- `onlyBuiltDependencies` restricts postinstall to `esbuild` and `@ast-grep/cli`

## Workspace Import Pattern (#packages/\*)

Root `package.json` defines Node.js subpath imports:

```json
{"imports": {"#packages/*": "./packages/*"}}
```

This must be resolved in 4 places:

1. **TypeScript** — via `"paths"` in tsconfig.json (generated by `packages_tsconfig()`)
2. **Node.js runtime** — via root `package.json` `"imports"` field
3. **Rolldown bundler** — via `resolve.alias` in `tools/rolldown-bundle.ts`
4. **Vitest/esbuild** — via esbuild plugin in `tools/esbuild-packages-resolve.mjs`

For Bazel sandbox contexts (vitest, ts_run_binary), the root `package.json` must be available:

- vitest: `data = ["//:package.json"]` + `no_copy_to_bin = ["//:package.json"]`
- ts_run_binary: `//:root_package_json` (copy_to_bin output) in `srcs`

## Linting & Formatting

| Tool       | Scope                  | Config                      |
| ---------- | ---------------------- | --------------------------- |
| oxfmt      | JS/TS/CSS/JSON/YAML/MD | `.oxfmtrc.json`             |
| oxlint     | JS/TS linting          | `.oxlintrc.json`            |
| ast-grep   | Structural lint        | `sgconfig.yml`, `sg-rules/` |
| buildifier | Bazel files            | Built-in                    |
| rustfmt    | Rust files             | Built-in                    |
| commitlint | Commit messages        | `.commitlintrc.mjs`         |
| syncpack   | Dependency versions    | `.syncpackrc.json`          |

## Git Hooks (lefthook)

Pre-commit hooks run sequentially:

1. `format-rust` — rustfmt on staged .rs files
2. `format-package-json` — syncpack fix + format
3. `format-oxfmt` — oxfmt on staged files
4. `format-starlark` — buildifier on BUILD/bzl files
5. `lint-staged` — oxlint --fix
6. `lint-ast-grep` — ast-grep scan on staged .ts/.tsx files
7. `gazelle` — regenerate BUILD files
8. `bazel-lock-regen` — bazel mod deps

Commit-msg hook: `commitlint` validates Conventional Commits format.

## CI/CD (GitHub Actions)

| Workflow               | Trigger               | What it does                             |
| ---------------------- | --------------------- | ---------------------------------------- |
| `test.yml`             | PR, push to main      | `bazel test //...` on Ubuntu + macOS     |
| `rust-cli-release.yml` | Tag `formatjs_cli_v*` | Cross-platform Rust binary release       |
| `release.yml`          | Push to main          | `bazel build :dist` then `pnpm publish`  |
| `prerelease.yml`       | Manual                | `lerna version` (independent versioning) |
| `website.yml`          | Manual/push           | Deploy docs site                         |
| `verify-hooks.yml`     | PR                    | Verify lefthook hooks + commitlint       |

## Common Commands

```bash
pnpm install              # Install deps
pnpm t                    # syncpack lint + oxlint + ast-grep scan + bazel test //...
bazel build //...         # Build everything
bazel test //packages/intl-localematcher:unit_test --test_output=all  # Test specific package
bazel run -c opt //crates/icu_messageformat_parser:comparison_bench   # Rust benchmarks (must use -c opt)
pnpm format               # runs lefthook pre-commit --all-files
bazel run //docs:serve    # Run docs site
bazel query 'kind(test, //packages/...)'  # Query test targets
```
